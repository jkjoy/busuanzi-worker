1. 快速接入：busuanzi 最大兼容

在任意网页中加入下面代码即可。脚本会自动把统计请求发送到 https://busuanzi.loliko.cn/api/count，同时按当前页面域名和 URL 记录统计。

<script async src="https://busuanzi.loliko.cn/busuanzi.pure.mini.js?v=3"></script>

本站总访问量：<span id="busuanzi_value_site_pv">0</span>
本站访客数：<span id="busuanzi_value_site_uv">0</span>
本文访问量：<span id="busuanzi_value_page_pv">0</span>

2. 自定义 ID 接入

如果你不想使用 busuanzi 的 DOM ID，也可以使用 eo_count_* ID。

<script async src="https://busuanzi.loliko.cn/count.js?v=3"></script>

本站总访问量：<span id="eo_count_site_pv">0</span>
本站访客数：<span id="eo_count_site_uv">0</span>
本文访问量：<span id="eo_count_page_pv">0</span>

3. EdgeOne Makers 部署说明

  1. 在 EdgeOne Makers 创建项目并关联仓库。
  2. 在 KV Storage 中创建命名空间，例如 busuanzi。
  3. 把 KV 命名空间绑定到当前项目，变量名设置为 my_kv。
  4. 部署后确认以下地址可访问：/count.js、/busuanzi.pure.mini.js、/api/count。

4. API 调用格式

脚本会自动生成 JSONP 请求；你也可以手动测试：

https://busuanzi.loliko.cn/api/count?site=example.com&page=/hello&vid=test&callback=cb

返回示例：

cb({"site_pv":1,"site_uv":1,"page_pv":1})

5. 常见问题排错

  • 如果控制台请求到了 https://你的博客域名/api/count，说明使用了旧脚本，请给脚本 URL 加 ?v=3 强制刷新缓存。
  • 如果 /api/count 返回 404，检查 Makers 是否识别 edge-functions/api/count.js。
  • 如果返回 545，优先检查 KV 绑定变量名是否是 my_kv，以及 KV key 是否只含字母、数字、下划线。

6. 统计说明

  • Site PV：站点总访问量。
  • Site UV：基于本地 visitor id 的站点访客数。
  • Page PV：当前页面访问量。
  • KV 是最终一致性存储，极高并发下短时间内可能存在轻微误差。
